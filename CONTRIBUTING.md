# Contributing to OpenRoster

Thank you for contributing to OpenRoster.

1. Open an issue before starting a substantial change.
2. Create a focused branch and keep the change narrowly scoped.
3. Run `npm run lint`, `npm run typecheck` and `npm test`.
4. Add tests for new calculation, authorization or tenant-isolation logic.
5. Submit a pull request describing the problem, solution and verification.

Contributions are published under `AGPL-3.0-only`.
