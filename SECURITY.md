# Security Policy

## Reporting a vulnerability

Do not report vulnerabilities through public issues. Use the repository's
private GitHub Security Advisory feature or the security contact listed by the
maintainers.

Include affected versions, reproduction steps, impact and a possible fix when
available. Reports will be acknowledged, prioritized and disclosed in a
coordinated manner.

## Supported versions

During the MVP phase, only the current version on the `main` branch is
supported.

## Operator responsibilities

Self-hosting operators are responsible for TLS, strong secrets, database
backups, SMTP configuration, timely updates and infrastructure access control.
